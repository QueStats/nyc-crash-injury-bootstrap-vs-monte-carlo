ST453 Final Project: Bootstrap Inference for NYC Crash Injury Severity
Quincy Cornish

1. Data Description

This project analyzes the New York City Motor Vehicle Collisions -- Crashes data
set, a public data set maintained through NYC OpenData. Each row represents a
police-reported motor vehicle collision. The data include crash time, borough,
injury counts, contributing factors, and vehicle type codes. The response
variable is NUMBER.OF.PERSONS.INJURED, which records the total number of persons
injured in a crash.

The final project uses the same real data source and model structure as the
midterm project. Six variables are retained from the raw file: CRASH.TIME,
BOROUGH, NUMBER.OF.PERSONS.INJURED, CONTRIBUTING.FACTOR.VEHICLE.1,
VEHICLE.TYPE.CODE.1, and VEHICLE.TYPE.CODE.2. Rows with missing or blank values
in key fields are removed, and a random sample of 20,000 crashes is drawn using
set.seed(1). After constructing the model matrix and removing incomplete rows,
the analysis contains approximately 19,999 observations and 15 model columns,
including the intercept.

Data citation: NYC OpenData. Motor Vehicle Collisions -- Crashes. New York City
Department of Transportation. Available at:
https://data.cityofnewyork.us/Public-Safety/Motor-Vehicle-Collisions-Crashes/h9gi-nx95

2. Variables of Interest

The outcome of interest is the number of persons injured per crash. This is used
as a crash severity measure. The main goal is to estimate how expected injury
counts vary with observable crash characteristics.

The predictors include a Night indicator, a MultiVehicle indicator, borough
indicators, contributing-factor indicators, and vehicle-type indicators. Night is
coded as 1 for crashes between 9 PM and 5 AM. MultiVehicle is coded as 1 when a
second vehicle type is recorded. Borough indicators include Brooklyn, Manhattan,
Queens, and Bronx, with Staten Island as the reference category. Contributing
factor indicators include driver inattention/distraction, following too closely,
failure to yield right-of-way, and unspecified factor. Vehicle indicators include
sedan, SUV, pickup, and motorcycle.

The main relationships studied are whether nighttime crashes are associated with
higher injury counts, whether failure-to-yield crashes are associated with higher
severity, and whether motorcycle crashes are associated with higher injury counts
after controlling for other variables.

3. Statistical Model

The statistical model is an ordinary least squares regression model. Let Y_i be
the number of persons injured in crash i, and let X_i be the vector of observed
crash characteristics. The model is

Y_i = X_i^T beta + U_i,

where beta is the vector of regression coefficients and U_i is an error term with
mean zero. The model estimates the conditional mean E(Y_i | X_i). This model is a
simplification because the response is a nonnegative count with many zeros. A
Poisson, negative binomial, or zero-inflated model could be more realistic.
However, the linear model gives an interpretable approximation to the conditional
mean and connects directly to the midterm project.

4. Estimation Procedure

The coefficient vector beta is estimated by ordinary least squares. The estimator
minimizes the squared error ||Y - X beta||^2. Computationally, the code uses
qr.solve(X, y), which solves the least squares problem through QR decomposition
in base R. This avoids explicitly computing a matrix inverse for the coefficient
estimate.

After fitting the model, fitted values, residuals, residual standard deviation,
R-squared, adjusted R-squared, standard errors, t-statistics, p-values, and
parametric confidence intervals are computed. These parametric results provide
the comparison point for the bootstrap inference.

5. Bootstrap Strategy

The final project replaces the midterm parametric simulation with a bootstrap
study. Since the model is a linear regression with fixed predictors, the selected
bootstrap strategy is a residual bootstrap. The design matrix X is held fixed,
and the empirical residuals from the fitted model are resampled to create new
response vectors.

The residual bootstrap follows these steps:

1. Fit the OLS model to the real data and compute fitted values and residuals.
2. Compute leverage values from the diagonal of the projection matrix.
3. Standardize the residuals using r_i = u_i / sqrt(1 - h_i).
4. Center the standardized residuals by subtracting their mean.
5. Sample residuals with replacement.
6. Create a bootstrap response Y* = X beta_hat + u*.
7. Refit OLS to (X, Y*) and store the bootstrap coefficient estimates.
8. Repeat the process N times.

This strategy matches the model structure because the fitted linear mean is kept
fixed while the error distribution is approximated by the empirical residual
distribution. It also avoids assuming Gaussian errors, which is the main contrast
with the midterm parametric simulation.

6. Number of Bootstrap Samples

The study uses N = 1,000 bootstrap samples. This is large enough to produce
stable histograms, standard error estimates, and percentile confidence intervals,
while still being practical to run in base R on a sample of about 20,000 crashes.
Using fewer bootstrap samples would make the tails of the bootstrap distribution
less stable, which would directly affect 95% confidence intervals.

7. Bootstrap Inference

For each coefficient, the bootstrap distribution is summarized using the mean,
standard deviation, percentile confidence interval, and a two-sided bootstrap
p-value. The percentile confidence interval uses the 2.5th and 97.5th percentiles
of the bootstrap estimates. The bootstrap p-value is computed as twice the
smaller proportion of bootstrap estimates falling on either side of zero.

The main coefficient of interest is Night. In the midterm project, the Night
coefficient was positive, indicating that nighttime crashes were associated with
higher expected injury counts after controlling for other variables. The final
project checks whether this conclusion remains under bootstrap inference. If the
bootstrap confidence interval for Night is entirely above zero, then the
bootstrap analysis supports the same conclusion as the parametric t-interval.

8. Comparison to Parametric Midterm Inference

The midterm project used parametric inference based on the OLS standard error and
t-distribution approximation. It also used a parametric simulation where new
responses were generated from Y = X beta_hat + epsilon with Gaussian errors. The
final project instead uses the empirical residual distribution. Therefore, the
bootstrap inference is less dependent on the Gaussian error assumption.

The key comparison is whether bootstrap standard errors and confidence intervals
are close to the parametric standard errors and confidence intervals. If they are
similar, this suggests the parametric inference is reasonably stable for this
model. If they differ substantially, then the parametric assumptions may be too
strong for the crash injury data.

The expected result is that the main signs and conclusions should remain similar,
especially for large effects such as FailureToYield and Motorcycle. Some
coefficients with weaker effects may have bootstrap intervals that are wider or
more centered around zero.

9. Broader Implications

The fitted model suggests that crash injury severity is related to observable
features of the crash. Positive coefficients for Night, FailureToYield, and
Motorcycle are plausible. Nighttime driving can involve reduced visibility and
fatigue. Failure-to-yield crashes can reflect dangerous intersection conflicts.
Motorcycle crashes can lead to greater injury severity because riders have less
physical protection.

These implications are reasonable, but they should not be interpreted as causal
claims. The data are observational. The model estimates conditional associations,
not controlled experimental effects. Still, the results can identify patterns
that are relevant for traffic safety and further analysis.

10. Limitations

The main limitation is that the response is a count variable with many zeros, but
the model is linear. This can lead to unrealistic fitted values and limited
predictive power. A count model such as Poisson regression, negative binomial
regression, or a zero-inflated model may better match the outcome distribution.

A second limitation is omitted-variable bias. Important factors such as speed,
weather, impairment, seatbelt use, road design, traffic volume, and impact angle
are not included. If these omitted factors are correlated with included
predictors, the regression coefficients may partly reflect omitted information.

A third limitation is predictor simplification. Contributing factors and vehicle
types are reduced into a small set of binary indicators. This improves
interpretability but loses information.

Finally, the bootstrap procedure evaluates uncertainty around the fitted linear
model. It does not prove that the linear model is the correct structural model
for crash injury severity. The bootstrap improves inference relative to a purely
Gaussian parametric simulation, but it still depends on the chosen regression
framework.
